Stack
01
CLAUDE.md § Stack
## Stack
HTML:    Vanilla — no framework (React/Vue/Svelte forbidden)
CSS:     Custom properties + utility classes — no Tailwind
JS:      Vanilla ES6 modules — no jQuery, no bundler
Icons:   Tabler Icons (CDN) — ti-* class names
Fonts:   Roboto (body) + Roboto Mono (code/data)
Charts:  Chart.js 4 — already loaded in base layout
Hosting: Static files — no server-side rendering needed
ℹ
Why vanilla? The client team maintains this post-handoff. No build toolchain means any dev can open a file, edit it, and see results. Resist the urge to add npm or a bundler — it's a deliberate constraint.

CLAUDE.md § Color corrections
## Color correction reference (WCAG AA failures — fix these)
FAIL: --text-secondary (#6B7280) on white → ratio 4.48 (need 4.5)
Fix:  use #636973 for body text on white backgrounds

FAIL: --color-warning (#F59E0B) as text on white → ratio 2.88
Fix:  use #92620A for warning text, #F59E0B for bg/icon only

FAIL: --color-accent (#0EA5A0) as small text on white → 3.01
Fix:  use #0A7A77 for accent text links on white surfaces

Always check new colour combinations at webaim.org/resources/contrastchecker

buttons
## Buttons
Classes: .btn-primary  indigo bg, white text — main CTA
         .btn-ghost    border only, transparent bg — secondary
         .btn-danger   red bg, white text — destructive actions
         .btn-link     no border/bg, teal text — inline actions

Sizes:   .btn-sm  32px h · 12px pad  (table row actions)
         .btn-md  40px h · 16px pad  (default — most forms)
         .btn-lg  48px h · 24px pad  (primary page CTAs)

Rules:
- border-radius: var(--input-radius) on all buttons
- Always include a loading state (spinner icon, disabled)
- Icon-only buttons need aria-label
Never create one-off button styles — use these four only

segmented control

Data & state management
06
CLAUDE.md § Data & state
## Data & state management
State pattern: plain JS module with exported state object
// /js/state.js
export const state = { patients: [], filters: {}, loading: false }
export function setState(patch) { Object.assign(state, patch); render() }

Data fetching:
- All API calls in /js/api.js — never inline fetch in HTML
- Always handle loading, error, empty states explicitly
- Mock data in /js/mock-data.js for dev — swap to API in prod
- Paginate all lists: default 25 per page

Persistence:
- localStorage: user preferences (theme, sidebar collapsed)
- sessionStorage: filter state, active tab
Never store patient/clinical data in localStorage

Forms & validation
08
CLAUDE.md § Forms & validation
## Forms & validation
Input spec: height 40px | pad 0 12px | radius 8px
            border 1px --border → --color-primary on focus
            outline: none (use border change for focus, not outline)

States:
  Default:  border --border
  Focus:    border --color-primary, box-shadow 0 0 0 3px rgba(91,91,214,0.15)
  Error:    border --color-danger, error message below (12px red)
  Success:  border --color-success (post-submit only)
  Disabled: bg surface-2, opacity 0.6, cursor not-allowed

Validation timing:
  Show errors: on blur (not on keystroke, not on submit only)
  Clear errors: on focus re-entry
  Required fields: asterisk (*) in label, not placeholder

Labels: always above input (never floating/placeholder-as-label)
Error messages: below input, --color-danger, 12px, icon prefix (ti-alert-circle)
Helper text: below input, --text-muted, 12px (shows before error occurs)

Date inputs: always use our custom DatePicker (z-index:9999)
             never use native <input type="date">

## Modals & overlays
Backdrop:  rgba(0,0,0,0.4) | z-index var(--z-modal)
Container: white | radius 16px | max-w 560px | pad 32px
           centre of viewport (flexbox on backdrop)

Standard modal:
  Header: title (16px 500) + close × icon (top-right)
  Body:   scrollable if content > 60vh
  Footer: right-aligned buttons (ghost + primary)

Confirmation modal:
  Max-w: 400px | Icon centred (40px, coloured by severity)
  Message: centred, 14px
  Buttons: full-width stacked (danger first, ghost below)

Success modal:
  Green checkmark (56px animated) | centred content
  Single "Done" or "View details" button

Side panel (detail panel):
  Slides in from right: width var(--panel-w) = 400px
  Full height | bg white | left border | z-index var(--z-modal)
  Header: title + close × | Body: scrollable | Footer: actions
  ESC key closes | backdrop click closes

Behaviour (all modals):
- Trap focus inside modal when open

Accessibility

CLAUDE.md § Accessibility — WCAG AA required
## Accessibility (WCAG AA — minimum standard)
Contrast:  4.5:1 normal text | 3:1 large text (18px+ or 14px bold)
Focus:     All interactive elements need visible focus ring
           box-shadow: 0 0 0 3px rgba(91,91,214,0.4)
           Never use outline:none without a custom focus indicator

Semantics:
  Use <button> for actions, <a> for navigation
  Forms: always <label for="id"> — no placeholder-only labels
  Tables: always <th scope="col"> headers
  Modals: role="dialog" aria-modal="true" aria-labelledby="title-id"
  Toggle: role="switch" aria-checked="true/false"
  Alerts: role="alert" for errors, role="status" for success

Keyboard:
  Tab: moves through all interactive elements in logical order
  ESC: closes modals, dropdowns, side panels
  Enter/Space: activates buttons and toggles
  Arrow keys: navigates within radio groups, tabs, menus

Images: always alt text | decorative images alt=""
Motion: respect prefers-reduced-motion for animations
✕
Healthcare requirement: This is a clinical application. Accessibility is non-negotiable — a user who cannot access a patient record because of a missing aria-label is a real patient care risk. Every component must be keyboard-navigable.

Usability heuristics
20
CLAUDE.md § Usability heuristics
## Usability heuristics (apply to every component)
1. System status visibility
   Loading states for every async action (never silent waits)
   Toast for every save/delete/create action
   Progress bar for multi-step flows

2. Error prevention over error recovery
   Disable submit until required fields are valid
   Confirmation modals for all destructive actions
   Show constraints before user hits them (e.g. char count)

3. Recognition over recall
   Always show current filter state in filter bar
   Breadcrumb or page title always visible
   Selected state always visually distinct from hover state

4. Flexibility & efficiency
   Keyboard shortcuts for power users (document them)
   Bulk selection + bulk actions on tables
   Filters persist across page reload (sessionStorage)

5. Aesthetic and minimalist design
   Every element on screen must earn its place
   Secondary actions live behind "..." menus
   Empty cells show "—" not a blank

6. Clinical-specific rules
   Patient names: always "Last, First" format in tables
   Dates: "Nov 12, 2025" — never MM/DD/YY (ambiguous)
   Times: 12-hour with AM/PM — clinical staff expect this

CLAUDE.md § Animation & motion
## Animation & motion
Principle: motion communicates, it doesn't decorate

Durations:
  --dur-fast:   150ms  micro-interactions (hover, toggle)
  --dur-base:   250ms  component transitions (modal, panel)
  --dur-slow:   400ms  page-level transitions (rare)

Easing:
  --ease-out:   cubic-bezier(0,0,0.2,1)   elements entering
  --ease-in:    cubic-bezier(0.4,0,1,1)   elements leaving
  --ease-both:  cubic-bezier(0.4,0,0.2,1) repositioning

Allowed animations:
  Modal enter:  translateY(8px)→0 + opacity 0→1, 250ms ease-out
  Side panel:   translateX(400px)→0, 250ms ease-out
  Toast enter:  translateY(16px)→0 + opacity 0→1, 200ms ease-out
  Skeleton:     shimmer gradient, 1.5s infinite
  Toggle:       thumb translateX, 200ms ease-both
  Tab switch:   border-color + bg, 150ms ease-out

Never animate width/height — use transform:scale instead
Never use animation for content that changes data
Always wrap in: @media (prefers-reduced-motion: no-preference)

Known bugs — never replicate
22
Critical
#bug-01
No box-shadow on table rows
Adding box-shadow to .table-row creates a z-index conflict with the sticky thead. Shadow bleeds through the header on scroll at any scroll position.
Fix: background-color change on hover only — no shadow.
Critical
#bug-02
No overflow:hidden on .card
Clips dropdown menus, DatePickers, and tooltips that render beyond the card boundary. Looks fixed but breaks sub-components.
Fix: keep overflow:visible (default). Handle radius with border alone.
Warning
#bug-03
DatePicker z-index must stay at 9999
Sidebar is z-index:100. Any DatePicker below that renders underneath. Already fixed — do not change when refactoring z-index values.
Fix: DatePicker wrapper always z-index:9999 — do not revert.
Warning
#bug-04
No flex gap inside .modal
gap shorthand in flex containers inside .modal breaks layout in IE11 (client mandate). Affects modal footer button rows specifically.
Fix: use margin-right / margin-bottom inside modals, not gap.
Note
#bug-05
Segmented control flicker on Safari
Transition on the active segment background flickers in Safari 16 when switching tabs quickly. Root cause: Safari GPU compositing bug with border-radius + transition.
Fix: add will-change:transform to .segment-active. Already applied in base CSS.

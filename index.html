Claude Skills

Everything you need to know about Claude Skills

A way to package repeatable design expertise into reusable instructions — defined once, applied consistently every time.

On this page
What are Claude Skills? Where Skills Work vs Fall Short Quick Decision Test UX Deliverables Skill Anatomy at a Glance How to Write a Skill Demo: Personas from SOW

What are Claude Skills?

Instead of writing detailed prompts again and again, Skills let you define once — and reuse forever. When you use a Skill, Claude:

📥
Loads a predefined instruction set
Like a mini system prompt — your rules, framework, and format are already in place.
🔁
Applies it consistently every time
No variation between sessions. Same structure, same standard, same quality bar.
📤
Produces predictable output
Every output follows the format you defined — ready to use, not ready to fix.

Where Skills Work vs Fall Short

Skills are powerful — but not universal. Knowing the boundary is as important as knowing the tool.

✅ Where it works
  • Repeatable tasks — UX audits, accessibility checks
  • Structured transformation — SOW → Persona, flows → journey map
  • Documentation-heavy work — design system docs, handoff specs
  • Checklist-driven processes — QA reviews, heuristic evaluations
🚫 Where it fails
  • Ambiguous problems — product strategy, vision definition
  • Insight-heavy work requiring judgment, not just formatting
  • Creative exploration — layout concepts, visual direction
  • Context-heavy decisions — tradeoffs, business vs user conflicts

Quick Decision Test

Not sure whether to use a Skill? Run through this quick check.

Use a Skill if…
✓ Task is repeatable
✓ A framework already exists
✓ Output can be structured
✓ The task is time-consuming
Avoid if…
✗ Needs deep judgment
✗ Problem is unclear or open-ended
✗ Creativity is the goal

Skill Anatomy at a Glance

Every well-written Skill is built from these six parts. The example below is for a Heuristic Evaluation Skill.

01
Purpose
"Evaluate a UI against Jakob Nielsen's usability heuristics and generate structured findings with severity scores."
02
Input
"Accepts UI screens (images), screen descriptions, or user flows. Avoid vague or incomplete inputs lacking UI context."
03
Framework / Method
"Use Jakob Nielsen's 10 usability heuristics. Assess the interface against each heuristic systematically."
04
Output Format
"For each heuristic, list: Issue · Evidence (from UI) · Impact on user · Severity (0–4) · Recommendation"
05
Rules / Constraints
"Do not invent missing context. Base all findings strictly on provided input. Avoid generic statements. Keep recommendations specific."
06
Evaluation Criteria
"Severity scale: 0 = No issue · 1 = Cosmetic · 2 = Minor · 3 = Major · 4 = Critical. Each issue must include a clear justification."

How to Write a Skill — Step by Step

Three steps from blank page to working Skill in Claude Code.

01
Write Skill Instructions
Works best when you want clarity, repeatability, and consistent outputs

Use the 6-part anatomy above. Structure each section with ## headers — Claude uses these to parse your instructions clearly.

skill-instructions.md
## Purpose
Evaluate a UI against Jakob Nielsen's 10 usability heuristics…
## Input
Accepts UI screens (images), screen descriptions, or user flows…
## Framework / Method
Use Jakob Nielsen's 10 usability heuristics as the evaluation framework…
## Output Format
For each heuristic, list: Issue · Evidence · Impact · Severity (0–4) · Recommendation
## Rules / Constraints
Do not invent missing context. Base all findings strictly on provided input…
## Evaluation Criteria
Severity: 0 = No issue · 1 = Cosmetic · 2 = Minor · 3 = Major · 4 = Critical
02
Create the Skill in Claude Code
Navigate to Skills in Claude Code and paste your instructions
1
Open Claude Code → click Customize in the sidebar
→
2
Go to Skills → click Create New
→
3
Paste your instructions → use ## headings for bold sections
→
4
Toggle switches on by default — Skill is ready to use
💡 Using ## to format section headings makes them bold in Claude Code, which significantly improves how Claude reads and applies your Skill.
03
Invoke the Skill
Type the skill name to apply it instantly

In Claude Code, type /skill-name followed by your input to trigger the Skill. Claude will immediately apply the predefined instructions.

Demo: Extracting Personas from SOW Documents

One of the most practical applications of Skills for design teams — turning business-focused SOWs into structured user personas.

Reality
  • UX starts from SOWs, not research
  • Documents are business-focused, not user-focused
  • User understanding is partial and implicit
→
Outcome (without Skills)
  • Personas built on assumptions
  • Gaps discovered too late
  • Misalignment between business goals and user needs
👉
What if we could extract and validate user understanding upfront?
A Persona Synthesis Skill takes raw SOW text as input, structures it against a persona template, explicitly flags assumptions, and surfaces gaps — before design work begins.

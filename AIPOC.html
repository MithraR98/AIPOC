🧠 AI for UX Design — Practical Workflow Guide

Deliverable → Input → Best Tool → How to Use It

⚠️ Before You Start (Most Important Insight)

👉 AI is only as good as your inputs

  • ❌ No inputs → Generic / misleading output
  • ⚠️ Weak inputs → Confident but wrong output
  • ✅ Clear inputs → High-quality results

🔍 Stage 1: Discovery

1. Heuristic Evaluation Report

You need: Screens or user flows

Use: Claude Skills

What it does: Reviews your UI against usability principles

👉 Tip: Paste screenshots + context → ask for usability issues + severity

2. Personas

You need: Interview insights or research notes

Use: Claude Code

What it does: Converts research into structured user personas

👉 Tip: Ask AI to include goals, frustrations, behaviors

🧩 Stage 2: Define

3. Information Architecture

You need: Feature/content list (PRD, SOW)

Use: FigJam

What it does: Organizes content into structure

4. User Flows

You need: Entry points + user goals

Use: FigJam

What it does: Maps how users move across the product

5. Task Flows

You need: Defined task steps

Use: FigJam

What it does: Breaks down one task into exact steps

6. Use Cases / Scenarios

You need: User goals + system actions

Use: Claude Code

7. Feature Prioritization

You need: Feature list

Use: Claude Code

8. Product Requirements

You need: SOW / BRD / stakeholder inputs

Use: Claude Code

9. Design Strategy Document

You need: Research + business goals

Use: Claude Code

🎨 Stage 3: Design

10. Low-Fidelity Wireframes

You need: User flows + feature list

Use: Figma Make

11. Mid-Fidelity Wireframes

You need: Low-fi or flows

Use: Figma Make

12. High-Fidelity Designs

You need: Wireframes + design direction

Use: Figma

👉 Important: Human control matters most here

13. Component Library

You need: Repeated UI patterns

Use: Figma + Claude Code

14. Design System

You need: Brand guidelines / UI patterns

Use: Figma + Claude Code

15. Accessibility Considerations

You need: Screens/components

Use: Claude Skills

16. Responsive Layouts

You need: Base screens

Use: Figma

17. Design System Documentation

You need: Final designs

Use: Claude Skills

18. Design QA Checklist

You need: Design system/patterns

Use: Claude Skills

✅ Stage 4: Validate & Deliver

19. Usability Testing Plan

You need: Flows + target users

Use: TBT (Not fully validated)

20. Test Scripts

You need: Tasks/scenarios

Use: TBT (Not fully validated)

21. Usability Reports

You need: Session notes/recordings

Use: TBT (Not fully validated)

22. A/B Test Hypotheses

You need: Problem + metrics

Use: Claude Code

23. Accessibility Audit Report

You need: Screens/product

Use: Claude Skills

24. Developer Handoff Specs

You need: Final designs

Use: Figma

🧭 Tool Cheat Sheet

  • Claude Code → Thinking, writing, structuring
  • Claude Skills → Evaluation, QA, audits
  • FigJam → Flows, collaboration
  • Figma Make → Early UI generation
  • Figma → Final design & control

📊 AI Reliability by Stage

  • Discovery → ⭐⭐⭐⭐ Strong for synthesis
  • Define → ⭐⭐⭐⭐ Great for structuring
  • Design → ⭐⭐⭐ Good for early drafts
  • Validate → ⭐⭐ Use cautiously

🚫 Where NOT to Use AI

  • Final design decisions
  • Complex workflows
  • Design system governance
  • Domain-heavy decisions

✅ Final Takeaway

👉 Use AI as:

  • A thinking partner
  • A speed booster
  • A checker for QA/accessibility

👉 Not as:

  • A decision maker
  • A replacement for design judgment
